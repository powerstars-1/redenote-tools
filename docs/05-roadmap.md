# 迭代路线

## Phase 0

文档先行准备阶段。

交付物：

- 项目范围定义
- 技术选型结论
- 架构设计
- 接口契约草案
- 面向部署的仓库结构

## Phase 1

基于 `Spider_XHS` 的小红书搜索与笔记详情服务封装。

交付物：

- 干净的服务骨架
- 搜索接口
- 详情接口
- 请求校验
- 结构化日志
- 健康检查接口
- 部署配置

## Phase 2

运维加固阶段。

交付物：

- 超时与重试调优
- 限流能力
- 请求去重
- 缓存策略
- 指标与追踪
- 日志脱敏校验

## Phase 3

扩展 `Spider_XHS` 能力。

候选功能：

- 二维码登录
- 创作者侧上传/发布
- 账号会话管理
- 评论获取
- 媒体下载流程

## Phase 4

演进为多渠道内容解析平台。

候选功能：

- 更多内容渠道
- 统一解析器抽象
- 平台专用 worker 模型
- 管理后台
- 配额与租户管理
