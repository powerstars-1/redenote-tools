# 项目范围

## 背景

本项目将作为自部署的后端服务运行，当前能力层以 `Spider_XHS` 为基础，后续会逐步扩展到其他内容渠道。

开发策略采用“文档先行”：先明确边界、契约和交付阶段，再进入编码实现。

## 产品定位

建设一个面向小红书内容检索的服务端能力层，可供上层调用方接入，例如：

- 扣子插件
- 内部工具
- 工作流系统
- 后续的管理后台或前端应用

## 第一阶段目标

第一阶段只包含两个核心能力：

- 按关键词获取搜索结果
- 通过笔记 URL 获取笔记详情

## 搜索能力需求

搜索接口需要支持以下输入参数：

- `keyword`：必填
- `note_type`：`default | image | video`
- `publish_time`：`default | 1d | 7d | 180d`
- `sort_by`：`general | latest | most_liked | most_commented | most_collected`
- `page_count`：整数，从第 1 页开始抓取前 N 页
- `cookie`：调用方传入，仅用于当前请求

## 笔记详情能力需求

详情接口需要支持以下输入参数：

- `url`：完整笔记 URL
- `cookie`：可选的调用方传入 Cookie

详情接口返回值需要提供标准化后的笔记元数据，并为后续排障或下载流程预留原始字段扩展能力。

## 第一阶段明确不做的内容

以下内容不属于第一阶段范围：

- 二维码登录流程
- 手机验证码登录流程
- 创作者侧上传/发布
- 评论抓取
- 媒体文件下载
- 多平台内容解析
- 账号管理界面
- 服务端持久化存储 Cookie

## 后续扩展方向

后续阶段可逐步增加：

- `Spider_XHS` 的二维码登录与上传发布能力
- 更安全的共享会话/凭据管理
- 其他内容渠道的解析器
- 任务队列、缓存、配额、可观测性能力
- 管理后台与运维控制能力
